Overview

This document describes dialogs that appear in printer configurator.

File: printconf.ycp
Main, so called complex dialog. The list of already configured printers is here, user can add, delete, edit printer and set printer as default.

File: printconf_ask_device.ycp
Ask user about settings for parallel port, usb, and serial port. User must enter port number. For serial port user must choose speed. User can test connection here.

File: printconf_ask_remote.ycp
Ask user about settings for remote connection. For tcp/ip forwarding queue, samba and novell printer requires user to specify remote host-name and remote queue name. For samba and novell user can enter username and password. For pre-filter queue user must specify tcp/ip queue - printconf can create new or already existing can be chosen.

File: printconf_device_confirm.ycp
This screen shows one auto-detected printer. If printer is fully auto-detected, user can configure it with detected manufacturer and model (button "next") or configure it manually (choose other manufacturer/model). User can skip printer.

File: printconf_features.ycp
Features of ascii printing. User can choose whether to print one page or two pages per sheet, if header will be printed, and portrait/landscape printing. User can choose encoding (between ISO-8859-[1|2] and system default).

File: printconf_filter.ycp
Filter options - settings of ghostscript. Paper size, color printing, dpi resolution, printer specific options. This dialog should be redesigned because it contains depended options. It means that change of value of some option can change some other options.

File: printconf_manuf_model.ycp
User chooses manufacturer and model. List of manufacturers are in left selection box and list of models are in right selection box. If there can be a problem with some model (e.g. GDI printer), information window is popped up. User can view an info about selected manufacturer and model.

File: printconf_spool_info.ycp
Users specifies name for printer queue. For tcp/ip fwd queue there is only on queue name (because input is not filtered and data are sent directly to remote queue). If new tcp/ip fwd queue is created for pre-filter queue, user enters also name for this queue. For all other printers user enters queue name. Name for ascii printing is derived from it by adding "-ascii".
